# Mneme 2.0 — LLM Wiki Refactor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **Goal:** Restore Mneme to the LLM Wiki thesis — OKF + tagged Markdown as the only source of truth, with `dream`/`search` as the only user intents — and ship 2.0.0 as a single skill zip with sqlite3 + FTS5 (no sqlite-vec / FastEmbed).
>
> **Architecture:** Dream (write) and Search (read) become the user-facing intents. Python CLI carries deterministic operations only (`init / lint / reindex / search`). `dream` is a read-only audit CLI; the actual writes happen in `SKILL.md` workflow (agent `Write`/`Edit` after explicit user approval). Atomic index rebuild uses `temp + fsync + os.replace` (preserves the v1.x contract). L2 (sqlite-vec + FastEmbed) is explicitly deferred to v2.1; v2.0 ships L0 + sqlite3 + FTS5.
>
> **Tech Stack:** Python 3.11+ stdlib (`sqlite3`, `tomllib`, `zipfile`, `argparse`), `PyYAML` (optional fallback), `pytest`. No L2 deps in v2.0.

## Global Constraints

- Target version: **`2.0.0`**. `__version__` in `skills/mneme/scripts/mneme/__init__.py`, frontmatter `version` in `skills/mneme/SKILL.md`, and zip file name must match exactly `mneme-2.0.0.zip`. `tests/test_version.py` enforces this.
- OKF v0.1 compliance remains the only must-confirm rule on bundle format (three MUSTs from `.research/upstream/OKF-SPEC.md:343-349`). Mneme does **not** add MUSTs to OKF; `tags` is a Mneme writer rule, not a protocol rule.
- Delivery model is **single skill zip** at `dist/mneme-2.0.0.zip`, unpacked to `~/.claude/skills/mneme/`. No wheel, no console command, no `mneme` import-by-name. `tests/test_release_layout.py` enforces this.
- L0 (index/tags/links/grep) and L1 (sqlite3 + FTS5, default on first `reindex`) are zero-dep. **L2 (sqlite-vec + FastEmbed) is NOT in v2.0**; it ships in v2.1. `mneme reindex --l2` and `mneme search --l2` do not exist in v2.0.
- User-facing verbs in SKILL/README/introduction: **`dream` and `search` only**. CLI `init/lint/reindex/search/dream` is for the agent; the user only says `dream` and `search`. CLI `dream` is read-only (audit/report).
- `mneme dream` never writes the bundle, never runs `git add -A`, never commits, never pushes. Writes after `dream`'s audit happen in the SKILL.md workflow via the agent's `Write`/`Edit` tools, after explicit user approval.
- Error philosophy: keep OKF v0.1 §9 tolerance consumer contract — one bad file never invalidates the bundle. External OKF missing `tags` is a WARN, not a failure.
- All Python source is in `skills/mneme/scripts/mneme/`. Skill entry file is `scripts/mneme.py`; call sites may also use `python3 -m mneme`. Tests live in `tests/`. PyYAML is optional; without it, lint still runs (warn-only strict mode).
- `pyproject.toml` MUST stay minimal: only `[tool.pytest.ini_options]`. CI matrix is **3.11–3.13 only** (no 3.10 cell; `config.py` requires 3.11+).
- CI must not run `python -m build --wheel`, `pip install -e '.[...]'`, or `pip install dist/mneme-*.whl`. Only `pytest -m 'not network and not compat'` plus zip integrity checks.
- Every task ends in an independently verifiable state — green tests for the unit touched, no orphan files, no broken links in introduction.

```python
from pathlib import Path
import os
from typing import Mapping


def resolve_bundle(
    *,
    config_dir: Path | None = None,
    env: Mapping[str, str] | None = None,
    cwd: Path | None = None,
) -> Path | None:
    env = env or os.environ
    cwd = cwd or Path.cwd()
    bv = env.get("MNEME_BUNDLE")
    if bv:
        return Path(bv)
    cd = Path(config_dir) if config_dir else Path(env.get("HOME", str(Path.home()))) / ".config" / "mneme"
    cp = cd / "config.toml"
    if cp.exists():
        bundle = _read_bundle_path_from(cp)
        if bundle:
            return bundle
    cur = cwd.resolve()
    for d in [cur, *cur.parents]:
        if (d / "index.md").exists():
            return d
    candidate = cwd / "wiki"
    return candidate if candidate.exists() else None
```

```python
def ensure_schema(conn):
    conn.executescript(
        """
        CREATE TABLE IF NOT EXISTS pages (
            id INTEGER PRIMARY KEY,
            path TEXT UNIQUE NOT NULL,
            type TEXT,
            title TEXT,
            description TEXT,
            tags TEXT,
            mtime REAL,
            body TEXT
        );
        CREATE VIRTUAL TABLE IF NOT EXISTS pages_fts USING fts5(
            title, description, tags, body,
            content='pages', content_rowid='id'
        );
        CREATE TRIGGER IF NOT EXISTS pages_ai AFTER INSERT ON pages BEGIN
            INSERT INTO pages_fts(rowid, title, description, tags, body)
            VALUES (new.id, new.title, new.description, new.tags, new.body);
        END;
        CREATE TRIGGER IF NOT EXISTS pages_ad AFTER DELETE ON pages BEGIN
            INSERT INTO pages_fts(pages_fts, rowid, title, description, tags, body)
            VALUES('delete', old.id, old.title, old.description, old.tags, old.body);
        END;
        CREATE TRIGGER IF NOT EXISTS pages_au AFTER UPDATE ON pages BEGIN
            INSERT INTO pages_fts(pages_fts, rowid, title, description, tags, body)
            VALUES('delete', old.id, old.title, old.description, old.tags, old.body);
            INSERT INTO pages_fts(rowid, title, description, tags, body)
```

```python
def search(query: str, db: Path, k: int = 10):
    conn = sqlite3.connect(str(db))
    rows = conn.execute(
        "SELECT p.path, p.title, snippet(pages_fts, 4, '|', '|', '…', 8) "
        "FROM pages_fts JOIN pages p ON p.id = pages_fts.rowid "
        "WHERE pages_fts MATCH ? ORDER BY rank LIMIT ?",
        (query, k),
```
