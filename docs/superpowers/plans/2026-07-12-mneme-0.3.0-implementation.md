# mneme 0.3.0 实施规格

> 上游依据：[1.0-readiness 评估](../reports/2026-07-12-mneme-1.0-readiness-assessment.md) · [v2.2 L2 索引规格](../specs/2026-07-11-mneme-search-design-v2_2.md) · [v2.1 skill 设计规格](../specs/2026-07-06-mneme-skill-design-v2_1.md)
> 适用基线：commit `e6d4b69`（v0.2.0）

## 1. 摘要与边界

本规格将 readiness 评估里的 Phase 0 / 1 / 2 拆成可 TDD 的红测与三个独立 PR。每个 PR 都以红测列为入口标准，commit message 与退出条件一并锁定。

**为什么只细化到 Phase 2：**

- Phase 3（workflow 端到端 acceptance）依赖校验器真起作用后才有意义，否则只能用现成的 false-positive 验收 = 等于验收了已知的错误行为
- Phase 4（retrieval benchmark）依赖真实 141 文档语料蒸馏成 OKF bundle——蒸馏本身在 v0.3.0 之后才能诚实做
- Phase 5（dream safety TDD）依赖 find_orphans 实现 + 真实相似度工作流 + git 原子事务测试，全部要在 Phase 1-2 稳定之后
- Phase 6（release gate）是 1.0 关卡，前置不达标时立 gate 只是把"未完成"标签贴得更整齐

把这四块提前细化 = 写出会被推翻的计划。v0.3.0 落地后另起一份计划即可。

## 2. 关键设计决策

### 2.1 YAML 解析策略 — 采用 A：producer 子集 + consumer 严格

**选择 A：** 生产端（`okflib.parse_frontmatter()` 输出 dict、`tools_helpers` 读 frontmatter）继续维持手写子集解析器（L1 写入路径零依赖）；消费端（`validate_okf.py` lint CLI、`okflib.validate_bundle()` 第三方调用）默认尝试 `import yaml`，若可用走 PyYAML 拒绝一切手写子集会放过的非法 YAML，若不可用降级为子集校验并 WARN `yaml-not-installed`。

**理由（三句）：**
1. 守住 L1 写入侧的"git-native + 零依赖"承诺——一个 LLM 写出 OKF bundle 不需要 pip 任何东西。
2. 报告 §Recommended TDD program §P0 要求的 red test 在 A 下能成立：装上 `mneme[validate]`（带 PyYAML）后跑校验，`type: [unterminated` 真的被 PyYAML 解析失败拒掉。
3. PyYAML 是 MIT、与项目同 License、加进 extras 不破坏分层（CLAUDE.md §硬约束 "L1 零依赖" 已经隐式允许 extras）。

**实现要点（不是代码，是约束）：**
- `pyproject.toml` 增加 `[project.optional-dependencies] validate = ["PyYAML>=6.0"]`
- `okflib.py` 顶部 `try: import yaml as _yaml; HAS_YAML=True\nexcept ImportError: HAS_YAML=False`，`validate_bundle()` 把 parse 与 verify 分开，verify 阶段走 PyYAML 重解析 frontmatter 块
- 不依赖 PyYAML 时 verify 阶段输出明确的 warning：`OKF-STRICT-VALIDATION-DISABLED: install 'mneme[validate]' for full YAML conformance`

### 2.2 交付模型 — 采用 CLI+skill

**选择 CLI+skill：** 把 `skills/mneme/scripts/*.py` 重构成 `src/mneme/` 真 Python 包，新增 `__main__.py` + `[project.scripts] mneme = "mneme.cli:main"`；SKILL.md 与 references/ 装进 `src/mneme/skill/` 并以 `package_data` 暴露；`pyproject.toml` 同时声明 `packages = ["mneme"]` 与 `[tool.setuptools.package-data]`。原 `skills/mneme/` 目录保留作为 symlink 安装形态（仍然 `ln -s` 进 `~/.claude/skills/mneme`），但其内容是符号链接到 `src/mneme/`。

**理由（三句）：**
1. SKILL.md 当前路径写法 `python3 skills/mneme/scripts/...` 在已 symlink 安装的 skill 目录下不存在——CLI+skill 用 `mneme init` 命令替换掉这个不一致，路径真收敛。
2. 报告 §P0 Delivery model 已经定性"当前 hybrid 是最不优雅的选择"；skill-first 单走意味着要回到 shell 包装一套，也不解决 dist 不可重现。
3. `[project.scripts]` + `package_data` 是 PyPA 标准件，`python -m build` 一行产物能与 `src/mneme/skill/` 字节对齐（Phase 2 red test 5.10）。

**实现要点：**
- 删除 `dist/` 手工复制；删除 `mneme.egg-info/`（每次 build 重新生成）
- `src/mneme/__init__.py` 暴露 `__version__ = "0.3.0"`
- `src/mneme/cli.py` 即现在的 `scripts/mneme.py`，加上 `if __name__ == "__main__": sys.exit(main())`
- `src/mneme/skill/` 软链自 `skills/mneme/`；build 时 wheel 用 `package_data` 把 skill 装进 `mneme/skill/`
- AGENTS.md / SKILL.md / workflow-*.md 中所有 `python3 scripts/...` → `mneme ...`（或 `python3 -m mneme`）

### 2.3 dream 处置 — v0.3.0 显式移除

**移除 dream：** SKILL.md §"Scenario: dream" 整章删除；`SKILL cn.md` 对应章节删除；`workflow-lint.md` 不引用 dream。`mneme.py`（即新 `mneme/cli.py`）不注册 `dream` 子命令。v0.3.0 后若恢复，必须先有：(a) Phase 5 retrieval benchmark 通过；(b) `find_orphans` + 真实相似度工作流通过红测；(c) 自动 commit 走 dry-run preview 模式 + 一份独立 safety TDD 清单。

**理由（三句）：**
1. 报告 §P0 Dream 列的三条 block（数学错、依赖缺失函数、`git add -A` 在解析前提交）任一条都是合并前事故——v0.3.0 没有"自动 dream"运行场景时，留尾巴纯增维护成本。
2. SKILL.md 描述的 dream 是 host-agent prose，不是 skill 真正可调用的工具；在 v0.3.0 范围内等于文档空白支票。
3. 恢复触发条件具体到可验证项，避免下一次 readiness 评估再列同样的 P0。

### 2.4 TOML 读写策略

**选择：** 读 = `tomllib`（Python 3.11+ stdlib）；Python 3.10 降级到 `tomli`。写 = `tomli_w`。删除 `tools_helpers.py` 的 ad-hoc `splitlines + split('=', 1)` parser。

**理由（三句）：**
1. `tomllib` 是 stdlib 解析器，与本项目"std 优先 + 可选 extras"的硬约束一致——基础安装零三方依赖，仅 extras 才拉 tomli。
2. `tomli_w` 是工业级 TOML writer（MIT，正确处理引号/反斜杠/控制字符转义），比手写稳——`test_bundle_path_with_quote`（§5.4）需要这种稳。
3. Python 3.10 仍是项目声明的 supports 版本（`requires-python = ">=3.10"`），graceful 降级避免硬弃。

**实现要点：**
- `pyproject.toml`: `dependencies = ["tomli_w>=0.4"]`；新增 extras `toml10 = ["tomli>=2.0"]`，运行时 `sys.version_info >= (3, 11)` 选 `tomllib`，否则 `import tomli as tomllib`
- 抽出 `mneme.config` 模块：`read_config(path: Path) -> dict`、`write_config(path: Path, data: Mapping)` 两个公开 API；`tools_helpers.resolve_bundle` 内部走 `read_config`
- §5.4、§5.5、§5.6 引号/中文/空格 round-trip 全部走真 TOML，不再用 splitlines 拼接
