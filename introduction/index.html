§ 1 · 初衷 — 为什么把 LLM Wiki 绑在 Agent 上

大多数知识工具让 LLM 每次 都从头检索、重新理解同一份资料。Mneme 换了一种思路：让 Agent 像维护源码一样，把原始资料逐步编译成互相链接的 Markdown 概念页。一次摄入、多次查询、持续修订——这是 Karpathy 在 LLM Wiki gist 里说的 "compile once, read many times"。

本地图——而不是云端 SaaS——是这件事的关键。它意味着：知识留在你手里的普通 Markdown 文件里，能用 git 一样 diff / blame / branch；Agent 仍能用它回答问题；规模上来之后再打开 L2 语义索引。

三块场景，它都更顺手

🌱 心理咨询 & 个人反思

把零散笔记、阅读摘录、对话要点分门别类连成你自己的网络。私密、可回看、长期积累；本地优先让表达更自由。

🗂️ 长期文档整理

把项目资料、外部文章、API 笔记按概念而不是日期组织；Agent 在你提问时只读它真的需要的那几页，不在 Prompt 窗口里反复重新理解。

👥 团队知识沉淀

Google 2026 年发布的 OKF v0.1 把 Wiki 标准化为带 YAML frontmatter 的 Markdown + 索引；Mneme 是它的一个本地优先、不依赖服务端的实现。

§ 2 · 安装 & 使用

A · 通过 skills.sh（首选）

一行命令，把 skill 安装到你的 Agent skills 目录：

npx skills add Scott1743/mneme

B · 从 GitHub 直接链接

git clone https://github.com/Scott1743/mneme.git ~/mneme
ln -s ~/mneme/skills/mneme ~/.claude/skills/mneme
# 现在 ~/.claude/skills/mneme/scripts/mneme.py 就是 CLI 入口。

C · 三步建立你的第一座 Wiki

# 1. 在任意目录建一个空 wiki（CLI 会创建 sources/、index.md、log.md）
python3 ~/.claude/skills/mneme/scripts/mneme.py init ~/my-wiki

# 2. 把要研究的资料（PDF 文本、网页存档、自己写过的 .md……）
#    放进 ~/my-wiki/sources/

# 3. 让 Agent 用一句中文开启摄入流程：
#    「用 mneme 把 my-wiki/sources/ 里的资料摄入进去，建立概念页。」

之后你想搜任何主题，让 Agent 直接读 Wiki：

# 构建 L2 语义索引（首次会联网下载 ~90MB 的 BGE 模型，之后离线可用）
python3 ~/.claude/skills/mneme/scripts/mneme.py reindex

# 自然语言检索
python3 ~/.claude/skills/mneme/scripts/mneme.py search "拖延到底在保护我什么" --limit 8
使用上的边界：你用 LLM Wiki 不是为了 取代 判断，是为了 替你记住 那些值得长期回看的判断。心理咨询方向请把它当成「私人索引」——诊断和处置始终交给专业人员。

§ 3 · 原理 — 它怎么工作

一个朴素的类比（写给不写代码的伙伴）

把一座 Wiki 想成一座你自己的小图书馆：

  • 原始资料（sources/） 是永远不动的高保真原件，相当于图书馆的善本库——Agent 只读不改。
  • OKF Wiki 是 Agent 写出来的互相参照的阅读卡片——每一张都带一个 type（Concept / Reference / Summary / Source）和时间戳。
  • index.md + log.md 是图书馆的主目录 + 入藏登记簿——让你和未来的你都能一目了然。
  • CLAUDE.md / SKILL.md 是馆里的工作守则——告诉 Agent 什么时候读、什么时候写、不要做什么。

三层架构

原始资料 sources/ 不可变的事实来源 │ ▼ OKF Wiki Agent 维护的 Markdown 概念网络 │ ▼ CLAUDE.md + SKILL.md 人与 Agent 共同维护的工作规约

一次 ingest 在背后做了什么

  1. 你丢一份原始资料进 sources/（PDF 文本、.md、网页存档……）。
  2. Agent 用 OKF 标准读取 frontmatter，把不变量留给 sources，把可蒸馏的概念写到 concepts/。
  3. Agent 更新本目录的 index.md（渐进式索引）和 log.md（日期前缀时间线）。
  4. mneme reindex 把所有概念按 L2 索引建好，搜索时只取相关页作答并附引用。
  5. mneme lint 周期性自检：OKF 硬约束 + 孤儿页 + 断链 + 缺页的重要概念——只告警不拒绝。

关键代码片段（两段，选自 skills/mneme/scripts/mneme/）

1) 创建 Wiki（cli.py 里 cmd_init 的 18 行核心——整页外壳就这么简单）：

def cmd_init(args: argparse.Namespace) -> int:
    bundle = Path(args.path)
    config = Path(args.config)
    bundle.mkdir(parents=True, exist_ok=True)
    (bundle / "sources").mkdir(exist_ok=True)
    (bundle / "sources" / ".gitkeep").touch(exist_ok=True)
    if not (bundle / "index.md").exists():
        (bundle / "index.md").write_text(
            '---\nokf_version: "0.1"\n---\n\n# Concepts\n', encoding="utf-8"
        )
    if not (bundle / "log.md").exists():
        (bundle / "log.md").write_text("# Directory Update Log\n", encoding="utf-8")
    _write_config(bundle, config)
    return 0

2) 解析 frontmatter（okflib.py 里 stdlib-only 实现，没有任何 YAML 依赖）：

_FRONTMATTER_RE = re.compile(
    r"\A---[ \t]*\n(.*?)\n---[ \t]*\n?(.*)\Z", re.S
)
_LOG_HEADING_DATE_RE = re.compile(r"^## (\d{4}-\d{2}-\d{2})\b")
# OKF §4 — `type` 必填，其余 (title/description/tags/timestamp)
# 是 recommended；未知键会被保留，未知 type 只告警不拒绝。

四个角色，就是用它做的四件事

📥 ingest

丢原始资料进来 → Agent 读、讨论、写摘要页、更新索引与时间线。

🔎 query

对 Wiki 提问 → 先读 index 定位 → 读权威页综合 → 标注每条 bundle 引用。

🩺 lint

周期性体检：找矛盾、过时论断、孤儿页、缺页的重要概念；容忍未知的 type 与可选字段缺失。

🔍 search

资料规模上来后，sqlite-vec + FastEmbed 本地索引接管检索——首次按需懒装。

底层 100% 本地：Python 标准库写成的 OKF 库 + 可选的 sqlite-vec / fastembed；没有数据库服务、没有云、没有构建步骤。能 cat 就能读，能 git clone 就能 ship。
