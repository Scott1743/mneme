§ 1 · 初衷

.mneme/index.db ← sqlite3 + FTS5 (默认) .mneme/cache/ ← agent 记忆;永不是真相源 wiki/ ← 你读什么;你 git 什么

大多数知识工具让 LLM 每次 都从头检索、重新理解同一份资料。Mneme 换了一种思路：让 Agent 像维护源码一样，把原始资料逐步编译成互相链接的 Markdown 概念页。

一次摄入、多次查询、持续修订——这是 Karpathy 在 LLM Wiki gist 里说的 compile once, read many times。

本地优先意味着：知识留在你手里的普通 Markdown 文件里，能用 git 一样 diff / blame / branch；Agent 仍能用它回答问题；规模上来时打开 SQLite FTS5 加速导航，索引永远是 可丢弃的。

§ 2 · 先和 Agent 聊

一行命令，把 skill 安装到你的 Agent skills 目录：

npx skills add Scott1743/mneme

选择版本

npx skills add 始终跟随仓库默认分支，也就是最新的 v3.0.0 语义版。要固定在这个零依赖版本，请直接下载 v2.0.1 的 skill zip。

v2.0.1 · 基础版SQLite FTS5，零第三方运行时依赖 v3.0.0 · 语义版最新主分支；可选语义检索能力

安装到 ~/.claude/skills/mneme/ 之后，重启会话即可。Mneme 没有独立窗口、没有常驻服务、没有命令行仪表——你和你的 Agent 聊天就是了。

dream 收集你读过的资料，把它们编译成本地的 OKF Markdown 概念页；Agent 写 page、写 link、写 tag，你说了算。遇到本地 PDF、DOCX 或 PPTX 不能直接读时，它只会调用你已经安装的转换器生成派生文本，从不自行安装软件。

search 在你的 wiki 里走——按 index / tags / 链接 / FTS5 召回；找不到就说没找到，不编。

你:    我刚读了几篇关于 OKF 的文章,帮我把它们收进我的 wiki。

Agent: 好。把文章路径发给我吗?我会创建一个新 wiki,按 OKF 协议写
       摘要页,互相链接,更新 index,最后跑一遍 lint。
你:    我三周前看过一篇东西,里面讲到 SQLite FTS5 怎么建倒排索引——
       帮我找回来。

Agent: 让我读你的 wiki 看看。
       ← 命中「SQLite 内部」「FTS5 vs LIKE」两组概念页
         它们都引用了一份来源笔记——给你看摘要片段与原始资料路径。

§ 3 · 原理 —— 知识不是每次都被重新检索

这是 Mneme 真正想讲的事：知识不是每次提问都被重新发现——它是被你的 Agent 编译进一棵你看得见、改得了、行得通的树。

想象一下你读过的所有笔记、论文摘录、对话笔记——它们被放在一个本地文件夹里，被 Agent 加上 YAML frontmatter，被指向相邻的概念页，被链接、被标记。你可以用 git 一样 diff 它，blame 谁改了哪一行，branch 出一个实验分支。所有你想得起来的「我三周前看的那个东西」都被存在那里，不是被重新检索。

RAG 每次都重新发现知识——不持久、不积累、不互相引用。LLM Wiki 把知识编译一次：以后每次提问都是在已经存在的东西上行走的，不是在空仓库里爬的。

Mneme 让这件事对个人和团队都变得轻：本地存储、纯 Markdown、零常驻、零云依赖。你用 Agent 维护它；Agent 写、Agent 链、Agent 审 lint，你只是和 Agent 聊。规模上来时打开 SQLite FTS5；想要语义召回就装 [optional] v2.1 才会出现的语义索引。索引是垃圾，你不需要它也能用——它只是帮你走得更快。

dream

把资料编译进 wiki——增量维护，默认先出预览，等你点头才落盘。

search

在已有的 wiki 里走——读 index / tags / 链接；大 wiki 打开 FTS5 召回。

lint

周期性指出矛盾、孤儿、漂移——只告警，不替你改。

§ 4 · 你为什么可能需要它

一份查得到、改得了、被 Agent 维护着的知识，比随时重新检索的 RAG 库更接近「思维的外接硬盘」。Mneme 想做的就是这件事——把它说出来的语言是 OKF v0.1，把它写出来的工具是一个站在你 Agent 旁边的轻量 skill。心理咨询方向请把它当成私人索引：诊断和处置始终交给专业人员。

v2.0 只带 L0(index / tags / 链接 / grep) 和 L1(SQLite FTS5)——零依赖、默认可用。语义索引层放在 v2.1，并且只是导航加速器，永不是事实源。

§ 5 · 朋友项目

朋友项目

森林密语 / 塔罗树洞(tarot-confessional)

想用 Mneme 沉淀一次塔罗解读或长会话里的体会?试试 森林密语——它是 Mneme 在符号性反思方向上的同源项目。森林密语负责温柔地完成抽牌与象征性解读(本地优先,本地抽牌码不携带心事原文),Mneme 负责把有价值的体会整理进本地知识 Wiki,让反复出现的母题在时间的另一面再次浮现。两边都坚持:

  • 本地优先 / Agent 驱动 / 可检查的开放文件
  • 不替任何产品下诊断,不替任何决定表态
  • 写入是显式的——你不主动「保存到 Wiki」,什么都不会被留下
