Mneme Research Protocol · Design 01

检索不是一个分数：
0.5、2.2 与 3.2 的受控比较

为 Mneme 3.2.0 预先定义一项可复现的比较实验。它把历史向量检索、当前零依赖 FTS5 与未来语义/混合检索放进同一证据链，回答“提升在哪里，代价是什么”。

状态：实验前设计日期：2026-07-15实验运行：待 3.2.0 完成产物：HTML + JSONL + 图表数据

01 / 研究问题

3.2 的语义能力，是否在真实中文资料上带来可证明的净收益？

不预设“向量一定优于全文检索”。本实验区分精确术语、改写表达与无答案请求，并同时测量检索质量、延迟、索引成本和可复现性。

核心约束。 三个系统必须使用同一份冻结语料、同一份概念页表示和同一套相关性标注。不得把更好的页面拆分、更多的资料或不同的 qrels 误当成检索器提升。

02 / 系统矩阵

三个明确角色，而非三个模糊版本号

历史语义基线

0.5.0

BGE `bge-small-zh-v1.5` 与当时向量检索路径。旧报告仅有 5 条手选 query；本次将尽可能以锁定依赖重新运行，而非直接引用历史分数。

词法/零依赖基线

2.2.0

SQLite FTS5 与 BM25 排序。用于量化精确术语、标题、标签、缩写和 ID 的可靠性，以及零依赖所换来的启动与资源优势。

候选系统

3.2.0

未来实现的 embedding、向量或混合检索。实验开始前写入实际模型、向量库、chunk、过滤和重排配置；不得在结果出来后改变主指标。

03 / 可比性证据链

先冻结输入，再运行检索器

A · corpus

从飞书 Markdown 导出建立只读快照；记录文件清单、SHA-256、总字符数、语言比例与许可/隐私边界。

B · representation

生成一份规范 concept-page corpus。三个系统读取同一份页面，排除 raw source 与系统文件的规则完全一致。

C · qrels

独立标注 query 与相关页面。至少两位标注者；记录分歧和裁决，不让检索结果反向塑造答案。

D · runners

每个系统输出同构 JSONL：query、rank、path、score、耗时、版本与配置哈希。固定 seed 与离线模型 revision。

E · analysis

由同一分析脚本计算指标、置信区间、显著性与图表。原始结果和派生图表均可追溯。

04 / 检索任务与标注

把 0.5 的五题保留为历史组，但不让它主导结论

query 组最低数量目的例子
历史回归5复核 0.5 报告中的已知 querygstack、银行回单、录音
精确词项10检验 FTS5 的优势边界产品名、缩写、标题、ID
短语与改写15检验 embedding 对非字面表达的收益“如何规范阅读流程”
跨表达/长问句10检验多概念、中文措辞变化与噪声带上下文的自然语言问题
无答案10检验是否错误地伪造相关性语料明确不覆盖的主题

目标为至少 50 条 query。每条给出分级相关性：2 = 直接回答，1 = 有用支持，0 = 不相关；主分析在 page level 进行，并单独报告 source level 聚合。

05 / 预注册指标

质量与成本同等重要

  • 主指标：nDCG@10，处理分级相关性。
  • 召回：Recall@1、@3、@5、@10。
  • 排序：MRR@10。
  • 拒答：无答案 query 的 false-positive rate。
  • 工程：索引时长、索引字节数、冷启动、P50/P95 查询延迟、峰值 RSS、模型下载与第三方依赖。

06 / 统计规则

不用单一漂亮数字取代不确定性

  • 以 query 为重采样单位，计算 bootstrap 95% CI。
  • 以同一 query 的系统结果做配对比较；主检验为 paired randomization test。
  • 报告效应量与 CI，不以 p < 0.05 单独作结论。
  • 对 0.5、2.2、3.2 的三组比较进行 Holm 校正。
Δ nDCG@10 = mean(nDCG@103.2 − nDCG@10baseline)

08 / 3.2 发布门禁

先定义何时值得引入复杂度

最低结论门槛

3.2 不需要在每一类 query 上击败 2.2；但发布报告必须证明其在“短语与改写 + 跨表达”集合上相对 2.2 有正向效应量及 95% CI，并说明精确词项是否退化。若总体收益不明显，报告仍可得出“FTS5 更适合默认路径”的结论。任何结论必须同时呈现代价：下载、依赖、索引时长和延迟。

09 / 运行后交付物

未来实验报告应放在 reports/experiments/

  • 一份自包含 HTML 报告，嵌入所有最终图表和方法版本。
  • 不可变 manifest.json：corpus hashes、qrels 版本、代码 commit、依赖锁定、模型 revision、硬件与操作系统。
  • 每个系统一份原始 results.jsonl，以及生成指标与图表的脚本输出。
  • 明确标记“历史 0.5 原始数字”和“本次受控复跑结果”；两者不可混用。
